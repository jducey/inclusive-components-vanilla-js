State and Style

Communicate two states, collapsed or expanded to a non-visual user with aria-expanded on the button. The button is the best place for the attribute since it is the control and is findable.

The content div can be collapsed with the hidden attribute. Alternatively, use a class with display none which can be toggled.

Note: We don't need role="button" because the element itself implies that role. Also, aria-controls may only work in JAWS and the content directly following the heading should be sufficient.

Style the button so it inherits all from the heading. Then add + and - svgs to indicated the interaction.
